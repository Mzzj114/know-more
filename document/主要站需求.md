# 主要站的需求

## 网站主页

- 这个页面会具有相对 url ""，也就是它是用户输入网站后直接访问的页面

信息传达
- 这个页面包含项目的名称
- 这个页面包含项目的简介
- 这个页面展示网站的功能，包括教程(一般性教程和案例),文档，论坛
- 这个页面直观地表达出网站的定位，即通过互动教学让用户入门提示词工程，同时通过文档与社区提供持续支持
- 这个页面直观地展示用户能学到的东西，以及是否应用提示词工程技巧的对比

外观表达
- 这个页面是网站的门户，应当相较于其它页面更为精致，富有更多动画效果。
- 这个页面的整体风格和网站其它部分一致

技术实现
- 这个页面主要是前端，几乎没有后端逻辑
- 这个页面包含一些前往其它页面的链接
- 这个页面会包含图片
- 这个页面会包含一些动画效果
- 这个页面的代码应当模块化且注释明确，因为出于其前端本质之后会需要频繁地修改
- 这个页面是继承全局 base template

## 提示词工程互动式教程 (Prompt Engineering Interactive Tutorial)

- 这个页面会具有相对 url "tutorial/"

结构 (Structure)
- 这个页面包含三个部分，分别是
    a. **教程目录 (Directory)**: 切换不同的教程主题。
    b. **内容展示区 (Content)**: 展示知识点、Markdown 文本及交互指引。
    c. **AI 交互区 (Playground)**: 模拟聊天界面，供用户实操测试提示词。

外观表达 (Visuals)
- 这个页面的整体风格和网站其它部分一致。
- 采用响应式布局：
    - **桌面端 (Desktop)**: 三栏并行排列（目录 | 内容 | 交互）。
    - **移动端 (Mobile)**: 目录收纳至抽屉式菜单；内容与交互区域上下垂直排列。

内容驱动与配置 (Data-Driven Configuration)
- **JSON 驱动**: 教程内容由 JSON 文件定义，实现内容与展现逻辑解耦。参考下文以及 `static/tutorials/clear-goal.example.json`。
- **协议扩展性与交互模型**: JSON 结构的 `interaction` 字段应支持丰富的互动逻辑：
    - `highlights`: **(UI 引导)** 给指定 CSS 选择器加高亮/呼吸灯，强制用户关注特定 UI 部件。
    - `chat`: **(预设对话)** 预填提示词并自动触发 AI 回复，展示标准案例。
    - `quiz`: **(知识测验)** 弹出单选/判断题，用户回答正确后方可进入下一步。
    - `fill_blank`: **(填空练习)** 在提示词模板中嵌入输入框，让用户补全关键指令。
    - `output_check`: **(结果达成)** 监测 AI 回复是否满足特定标准（如字数、关键词），作为通关条件。

技术实现与协作 (Technical Implementation)
- **Vue 渲染引擎**: 前端作为状态机，根据当前 `step` 索引从 JSON 数组中提取数据并响应式渲染。
- **状态同步**: 教程进度（当前 Slug 和 Step）应与 URL 同步，确保刷新页面或分享链接时能精确定位进度。
- **安全代理 (Proxying)**: 所有的 AI 交互必须经由后端 Django 接口转发处理。前端根据 JSON 中定义的 `chat` 类型请求后端，由后端注入 API Key 后调用大模型。(目前只写请求并返回错误信息即可，后端api日后会实现)
- **可维护性**: 保持渲染逻辑的清晰，确保未来可以轻松增加新的步骤类型（如：视频步骤、填空题等）。
- **Base Template**: 页面继承全局基础模板，保持导航栏等公共组件一致。
