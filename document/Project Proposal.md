# 项目提案：Know More

## 项目简介

Know More 是一个面向普及与提升「提示词工程（Prompt Engineering）」技能的非盈利导航与学习平台。用户可以学习提示词工程的知识，在制定某领域学习路线的案例中练习，并测试提示词。

用户可以在 Know More 的社区论坛中找到按任务/场景获取可直接复用的 prompt 模板。平台的目标是普及AI的高阶使用，把复杂而不稳定的AI使用技巧降维为可复制的模板和交互式教学，使更多人能更高效、可控地使用大模型学习自己想学习的知识。

---

## 需求清单

* 网站主页有对好提示词重要性的宣传和对项目的介绍，当用户在主页点击开始学习后，就会进入教程页面。

* 教程页面有多个悬浮窗和tooltip，指引用户完成代表规范提示词结构的表单。
* 用户在按教程写出“好”提示词后可以在使用内置LLM测试提示词，并查看输出。

* 用户可以访问并使用“设计学习路线”的案例教学，这个案例教学会引导用户来引导LLM为用户设计一份学习新知识的路线

* 用户可以访问文档，文档中有对提示词工程，LLM提示词技巧(如 meta-prompting) 更详细的文字介绍。

* 用户可以访问并使用论坛。用户可以在论坛中发帖，回帖，点赞，收藏，分享。
* 论坛鼓励用户分享提示词，并讨论提示词和LLM使用

* 当管理员需要管理站点时，就可以通过后台查看投稿、用户活跃度、模型测试日志和举报/审核流。

## 技术栈

* 后端

  * **Python 3.11+**, **Django 4.x**
  * **Django REST Framework** — 提供 REST/GraphQL API
  * 模型/AI 接入层：封装为服务（使用 OpenAI / Anthropic 的 API， 为用户设置使用限制）
* 数据库与搜索
  * 主数据库：**MySQL**
  * 全文搜索（可选，日后再说）
* 前端
  * **Vue** + **Element Plus** 
  * 使用传统 django template 渲染，不使用 vite 或前后端分离
* 部署与 infra

  * 容器化：**Docker + docker-compose**
  * WSGI/ASGI：**uwsgi + Nginx 反向代理**
  * CI/CD：GitHub Actions（自动测试、构建、部署）

## 网站结构层次（页面/模块概览）

我大概买不起很多二级子域名，所以就做成一个站点，然后用不同的路径来区分不同的站点。
下面的所有站点都使用 django app 来实现，其中管理后台使用 django 内置的 admin site。

* 主要站点
  * 首页（宣传，搜索，开始学习/进入教程 按钮）
  
  * 教程页面（悬浮窗和tooltip，指引用户完成代表规范提示词结构的表单）
  
  * 案例页面（“设计学习路线”的案例教学）
  
* 文档站点（使用 flat-file CMS 的思路）
  * 文档页面（对提示词工程，LLM提示词技巧(如 meta-prompting) 更详细的文字介绍）
  
* 论坛站点
  * 论坛页面（发帖，回帖，点赞，收藏，分享）
  
* 管理后台（仅管理员）

## 前端样式风格指导

* 设计定位：**“教育 + 工具”** — 既要可信（专业），又要友好（低门槛）。风格应当现代、简洁、有层次。
* 详见 theme.css
* 无障碍与国际化（日后再说）
  * 目前网站的文本请使用中文
  * 将来考虑支持英文或其它i18n

* 风格文档

  * 建议建立一份简短的 Design System（颜色、间距、按钮、表单样式、卡片规范）以便一致性和协作。
