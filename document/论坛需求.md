# 论坛需求（设计指南）

## 整体架构思路

- **核心功能**（发帖、列表）使用 Django template 渲染，保证 SEO 和首屏加载
- **交互功能**（点赞、收藏、实时通知）使用 Vue + Element Plus 增强体验
- **API** 使用 Django REST Framework 提供接口给前端调用

---

## 数据库设计

### 核心表结构

1. 分类表 (Category)
2. 帖子表 (Post)
3. 回复表 (Reply)
4. 点赞表（Like）
5. 收藏表（Favourite）

---

## 后端职责 (Django)

### 1. **数据模型层 (models.py)**
- 定义所有数据库表结构
- 设置字段的验证规则
- 定义 `__str__` 方法便于调试
- 使用 Django 的 `get_absolute_url()` 定义对象的规范 URL

### 2. **业务逻辑层 (services/ 或直接在 views)**
- **帖子服务**：创建、编辑、删除、置顶、锁定
- **回复服务**：嵌套回复处理、楼层计数
- **统计服务**：浏览量递增、回复数更新
- **通知服务**：@提及、回复通知（可异步）

### 3. **视图层 (views.py)**
由于使用 Django template 渲染，视图分为两类：

**A. Template Views (页面渲染)**
```python
# 论坛首页 - 显示分类和帖子列表
# 帖子详情页 - 显示帖子内容和回复
# 发帖页面 - 表单
# 编辑页面 - 表单
# 个人中心 - 我发的帖、我的收藏
```

**B. API Views (DRF, 用于动态交互)**
```python
# POST /api/posts/{id}/like/ - 点赞
# POST /api/posts/{id}/favorite/ - 收藏
# GET /api/notifications/ - 获取通知
# POST /api/replies/ - 提交回复 (带富文本/Markdown)
```

### 4. **表单验证 (forms.py)**
- 标题长度限制（如 5-200 字符）
- 内容防 XSS（使用 bleach 库清理 HTML）
- 频率限制（同一用户多久发一帖）
- 敏感词过滤（可扩展）

### 5. **权限控制**
- 使用 Django 的 `@login_required` 装饰器
- 自定义权限：只有作者能编辑自己的帖子
- 管理员权限：删帖、封禁用户
- 使用 Django Group/Permission 系统

### 6. **信号 (signals.py)**
```python
# 当回复创建时：
# - 自动更新帖子的 reply_count
# - 更新 last_reply_at
# - 向帖子作者发送通知

# 当用户被@时：
# - 创建通知记录
```

## 前端职责

### 1. **Django Template (基础页面)**

**使用 base.html 扩展**

**页面模板**

在需要的地方挂载 Vue 应用：

**点赞组件**
```vue

```

**收藏按钮**
```vue

```

**回复编辑器**

## 关键功能实现思路

### 1. **帖子列表页**
```
后端：
- 按分类筛选
- 排序（最新/最热/置顶优先）
- 分页（Django Paginator）
- 预加载作者信息（select_related）

前端：
- 展示帖子卡片（标题、作者、回复数、浏览量、最后回复时间）
- 分类筛选器
- 搜索框（简单关键词搜索）
```

### 2. **帖子详情页**
```
后端：
- 帖子内容 + 作者信息
- 回复列表（支持分页，每页 20 条）
- 楼中楼回复（递归查询或一次性加载）
- 当前用户的点赞/收藏状态

前端：
- Markdown 渲染（使用 marked.js 或类似库）
- 代码高亮（highlight.js）
- @用户高亮
- 点赞/收藏按钮（带计数）
- 回复表单（支持@某人）
```

### 3. **富文本/Markdown 编辑器**
```
方案 1: 使用 vditor 或 editor.md（轻量，支持 Markdown）
方案 2: 使用 Quill 或 TinyMCE（富文本，功能强）

后端需要：
- 接收 Markdown 或 HTML
- 清理危险标签（bleach）
- 存储到数据库
- 渲染时添加代码高亮
```

### 4. **通知系统**
```
数据库表：
- Notification
  - recipient (接收者)
  - actor (触发者)
  - verb ('liked your post', 'replied to you')
  - target (关联对象)
  - unread (是否未读)
  - timestamp

实现方式：
- 同步：创建通知记录，前端轮询
- 异步（进阶）：使用 Celery + Redis，WebSocket 推送
```

### 5. **防刷机制**
```
后端：
- 使用 Django cache (Redis) 记录用户操作频率
- 同一用户 60 秒内只能点一次赞
- 发帖频率限制（如 5 分钟一贴）
- IP 限制（可选）
```

## 目录结构建议

```
forum/
├── migrations/
├── models.py          # 数据模型
├── forms.py           # 表单验证
├── views.py           # 视图（Template View + API View）
├── urls.py            # URL 路由
├── serializers.py     # DRF 序列化器
├── services.py        # 业务逻辑封装
├── signals.py         # Django 信号
├── utils.py           # 工具函数（如 Markdown 渲染）
├── templatetags/      # 自定义模板标签
│   └── forum_tags.py  # {% render_markdown %} 等
└── templates/
    └── forum/
        ├── index.html
        ├── post_detail.html
        ├── create_post.html
        └── _reply_list.html  # 局部模板
```

## 开发顺序建议

1. **第一阶段：核心功能**
   - 数据库模型

   - 基础权限控制

2. **第二阶段：互动功能**

   - Markdown 渲染
   - 楼中楼回复
   - 搜索功能
   - 标签系统

   - Django Admin 配置
   - 审核流
   - 数据统计面板
